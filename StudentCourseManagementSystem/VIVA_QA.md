# Viva Questions and Answers

## 1. What is the title of your project?
The title of my project is Student Course & Performance Management System.

## 2. What is the purpose of this project?
The purpose of this project is to manage student details, course records, enrollments, marks, attendance, and grades in a digital system.

## 3. Which programming language is used in this project?
Java is used in this project.

## 4. Which GUI technology is used?
Java Swing is used to build the graphical user interface.

## 5. Which Java concepts are used in this project?
This project uses classes, objects, inheritance-friendly design, encapsulation, collections, file handling, and event-driven programming.

## 6. How is data stored in the project?
Data is stored locally in CSV files.

## 7. Why did you use CSV files instead of a database?
CSV files are simple to manage, lightweight, and suitable for a college-level desktop project without requiring database setup.

## 8. What are the main modules in your project?
The main modules are Student Management, Course Management, Enrollment Management, and Dashboard Analytics.

## 9. How are grades calculated?
Grades are calculated automatically based on the marks entered for each enrolled student.

## 10. What happens when a record is updated?
The application validates the data, updates the in-memory records, and saves the latest data back to CSV files.

## 11. What is the role of the service layer?
The service layer contains the business logic, validation rules, and data processing functions.

## 12. What is the role of the model package?
The model package contains the data classes such as Student, Course, and Enrollment.

## 13. What is the role of the persistence package?
The persistence package is responsible for loading and saving data using files.

## 14. What is the role of the UI package?
The UI package contains the Swing interface for interacting with the system.

## 15. What are the advantages of your project?
It is simple, user-friendly, lightweight, and demonstrates multiple Java concepts in one complete application.

## 16. What are the limitations of your project?
It does not use a database, does not support multi-user access, and currently does not have authentication.

## 17. How can this project be improved in the future?
It can be improved by adding login, database support, report export, search, filters, and chart-based analytics.

## 18. Why is this project useful for colleges?
It helps manage academic records in a structured way and reduces manual paperwork.

## 19. Is this project object oriented?
Yes, the project is object oriented because it is built using classes and separates data, logic, and UI into different layers.

## 20. Why did you choose this topic?
I chose this topic because it is practical, easy to explain in viva, and demonstrates both Java programming and GUI development.
